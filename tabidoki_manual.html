Chapter 0

使用要件・推奨環境

0-1 アクセス方法

GitHub Pages（ホスティング）

https://chi555.github.io/tabidoki/

上記URLをブラウザで開くだけで利用できます。インストール不要、アカウント登録不要です。

スマートフォンで利用する場合はブラウザのブックマーク・ホーム画面追加が便利です。

オフライン利用について：行程の閲覧・編集・走行モードはオフラインでも動作します。天気取得・施設ピッカーのオンライン更新はインターネット接続が必要です。
0-2 推奨ブラウザ・デバイス
📱 Android Chrome BEST

全機能が最も安定して動作します。縦向き固定にも対応。

🍎 iOS Safari OK

ほぼ全機能対応。ファイル保存は「開いたタブで長押し」操作が必要。縦向き固定は非対応。

💻 PC Chrome / Edge OK

Nexus5シミュレーション（DevTools）での動作確認済み。マウスドラッグで並び替え可能。

🦊 Firefox / 旧OS 非推奨

動作は確認していますが、一部UIが崩れる場合があります。

表示サイズ：スマートフォン（幅 375〜480px）に最適化されています。タブレット・PCではページ幅が最大 480px に制限されます。
0-3 通信要件
🌤️

天気予報取得 自動

地点の住所が設定されている場合、Open-Meteo / OpenStreetMap Nominatim / 国土地理院 API に接続して天気予報を自動取得します。地点ごとに通信が発生します。

🛣️

高速SA・PA データ更新 バックグラウンド

Overpass API から全国高速施設データを取得・更新します。キャッシュが24時間以内なら通信しません。フォールバックデータ（230件）がオフラインで常時利用可能です。

🏪

道の駅データ更新 バックグラウンド

同様に全国道の駅データを取得します。キャッシュが24時間以内なら通信しません。フォールバックデータ（464件）がオフラインで常時利用可能です。

🗺️

マップリンク タップ時のみ

「マップで確認」ボタンをタップすると Google Maps が別タブで開きます。タップしない限り通信は発生しません。

0-4 ストレージ・データ保存

💾 JSON ファイル保存（推奨）

💾 ボタンで行程データを .json ファイルとしてデバイスに保存します。次回のツーリング計画にも再利用できます。大切なデータは必ずここでバックアップしてください。

📝 走行記録保存

📝 ボタンで実着・実発・走行メモを含む走行記録を .txt ファイルとして保存します。ツーリング後の記録残しに使います。

注意：ブラウザの「閲覧データ消去」やプライベートブラウズ終了でデータが消える場合があります。セッションをまたいでデータを保持するには 💾 保存 を必ず実行してください。

Chapter 1

起動画面の見かた

1-0 起動直後の画面

起動すると「木枯らし紋次郎」風のスプラッシュ画面が表示され、本画面に遷移します。スプラッシュの主演名・話数・タイトルは 📜 ボタンでカスタマイズできます。

🗺️

データ未読み込み状態
「📂 データの読み込みをしてください」と表示されます。
📂 読込 で保存済みファイルを開くか、
✏️ 編集 から地点を新規追加してください。

初回起動時・データ未読み込み時の状態

1-1 現在時刻の表示

23:16

ヘッダー上部に常時表示されるデジタル時計（15秒ごと更新）。走行モード中は大きく表示されます。時刻の色は時間帯によって変化します。

🌅 明け方（5〜6時）: オレンジ　☀️ 日中（6〜17時）: 白　🌆 夕方（17〜19時）: オレンジ　🌙 夜間（19〜5時）: 緑
テーマ連動：☀️/🌙 ボタンでテーマを手動切替した場合、60分ごとの自動テーマ変更は停止します。手動設定はブラウザを閉じても保持されます。
1-2 ヘッダーボタン
🏍️

走行モード 走行中メイン

ツーリング中に使うビュー。現在地・次の目的地・天気・出発カウントダウンを大きく表示。スワイプまたは ◀前 / 次▶ で地点を切り替えられます。走行モード中はボタンが 📋 に変わります。

✏️

編集 行程作成

地点の追加・編集・並び替えを行います。走行モード中に押すと走行モードを終了して編集モードに切り替わります。

💾

保存（JSON） 計画データ

現在の行程データを .json ファイルとして保存。ファイル名は「旅刻_タイトル_日付.json」形式。地点が1件もない場合は保存できません。

📂

読込 計画データ

保存済みの .json ファイルを読み込みます。現在の行程は上書きされます（確認ダイアログあり）。

📋

共有テキスト

行程をテキスト形式に変換。クリップボードにコピーして LINE・メモへの貼り付けに使えます。

📜

タイトル設定

起動スプラッシュの演出をカスタマイズ。主演名・第◯話・サブタイトルを自由に設定できます。

☀️

テーマ切替 視認性向上

ダークモード（夜間・室内向け）と昼間モード（屋外・直射日光下向け）を切り替えます。6〜18時は自動で昼間モードになりますが、手動切替後は自動変更が停止します。

📝

走行記録保存 走行後

帰宅後に実着・実発・走行メモを1件以上入力するとボタンが amber色に点灯します。押すと計画時刻・実績時刻・差分・メモを含む .txt ファイルを保存します。

🗺️

サンプルデータ

神奈川〜山梨の日帰りツーリングサンプルを読み込みます。初めて使うときの動作確認に便利です。

1-3 中央エリア（未読み込み時）
🗺️

「編集から地点を追加してください」
📂 読込 で保存済みファイルを開くか、
✏️ 編集 → ＋ 追加 で地点を追加してください。
🗺️ サンプル でまず動作確認もできます。

Chapter 2

編集画面の使いかた

2-0 編集画面の全体像

✏️ 編集ボタンを押すと上部にフォームが展開し、下に地点一覧が表示されます。フォームで地点を追加・編集し、一覧で並び替え・削除・現在地設定ができます。

① ヘッダー直下

日程タブ（1日目・2日目…）と日付・ルートURL入力欄が表示されます。

② 地点追加フォーム

地点名・時刻・メモ・走行メモ、詳細（住所・給油・実績時刻）を入力するエリアです。

③ 地点一覧（タイムライン）

登録した地点が縦のタイムライン形式で表示されます。各地点をタップするとアクションメニューが開きます。地点間には移動時間が色分け表示されます。

自動保存について：地点の追加・編集・削除・並び替えはすべて localStorage に自動保存されます。ただしセッションをまたぐバックアップには 💾 JSON 保存 が必要です。
2-1 日程タブ

日程の切り替え

タブをタップして日程を切り替えます。タブには「1日目 / 3/25(日)」のように日付と曜日が自動表示されます。

日程管理エリア

日付・ルートURLの入力欄と、日程追加・削除ボタンが表示されます。

日付

ツーリング日

日付を設定すると天気の取得基準日になります。同一日付の登録は不可。日付を設定した日程は自動的に日付順に並び替わります。

ルートURL

Google Maps ルートなど

http / https で始まる URL を登録すると、走行モードで「🗺 ルートを開く」ボタンが表示されます。Google マイマップや Yahooカーナビのルートリンクを登録しておくと便利です。

日程追加：「＋ 日程追加」で最大7日程まで追加できます。日程削除：「🗑 削除」でその日程の全地点ごと削除されます（確認ダイアログあり）。
2-2 地点の追加フォーム
地点名 必須

地点の名前（最大50文字）

例：道の駅みつまた、ENEOS SS、足柄SA（上り）。ピッカーで選ぶと自動入力されます。

着時刻 任意

到着予定時刻（HH:MM）

設定すると走行モードで「あと○分」カウントダウンが表示されます。

発時刻 任意

出発予定時刻（HH:MM）

着時刻より後の時刻を入力してください。着→発の滞在時間が自動計算されてフォームに表示されます（例：(1時間30分)）。

⏱ 既存地点の発時刻を変更すると、以降の地点の時刻が差分だけ自動調整されます（カスケード）。

メモ 任意

計画メモ（最大200文字）

食事・注意点・立ち寄り先など。行程ビューと走行モードに表示されます。「DM」「danger」「注意」等のキーワードを含む場合は赤文字で強調表示されます。

走行メモ 任意

📝 走行後の記録メモ（最大200文字）

走行中や帰宅後に入力する記録用メモ。走行記録保存（📝）に含まれます。入力があると 📝 ボタンが amber 点灯します。

時刻順バリデーション：着時刻 ＞ 発時刻 になっている場合はエラーが表示され追加/更新できません。
2-3 入力補助ピッカー（高速 / 道の駅 / GS）

地点名入力欄の下にある3つのチップボタンをタップすると、施設名をリストから選んで地点名と住所を自動入力できます。

🛣 高速 🏪 道の駅 ⛽ GS
🛣 高速

高速SA・PA・IC

全国の高速道路SA・PA・ICを選択。フォールバックデータ230件を内蔵。オンライン時はOverpass APIで最新データを自動取得・補完します。選択すると地点名と都道府県が自動入力されます。

🏪 道の駅

全国道の駅

全国の道の駅を選択。フォールバックデータ464件を内蔵。「道の駅 ◯◯」形式で地点名が入力されます。都道府県名で検索可能。

⛽ GS

ガソリンスタンド チェーン

ENEOS・apollostation・コスモ石油など14チェーンを選択。選択すると「チェーン名 SS」が地点名に入力され、給油ポイントが自動でONになります。

検索のコツ：ピッカーを開くと現在の地点名入力内容が検索欄に自動入力されます。部分一致で絞り込みができます。
2-4 詳細セクション

「詳細（住所・給油・実績）」ボタンを押すと追加入力欄が展開されます。

住所 天気用

住所（最大100文字）

天気予報取得のための住所です。住所があると天気が取得されます。「神奈川県小田原市」のように都道府県＋市区町村程度で十分です。ピッカーで施設を選ぶと自動入力されます。

⛽ 給油

給油ポイントフラグ

タップでON/OFF切替。ONにすると行程ビュー・走行モードで ⛽ アイコンが表示されます。GSピッカーで選択すると自動でONになります。

実着 走行後

実際の到着時刻

帰宅後に実際に到着した時刻を記録。走行記録保存で計画との差分（±○分）が出力されます。

実発 走行後

実際の出発時刻

帰宅後に実際に出発した時刻を記録。実着・実発・走行メモのいずれかが入力されると 📝 ボタンが amber 点灯します。

2-5 地点タップ時のアクションメニュー

編集モードで地点をタップすると行内にアクションメニューが表示されます。

📍

現在地にする

その地点を「現在地」としてマークします。走行モードで現在地として表示されます。行程ビューでは▶ 現在地バッジが付きます。

✏️

編集

フォームにその地点のデータを読み込み、内容を編集できます。編集中はボタンが「更新」に変わります。

🗺

マップ

住所が登録されている場合のみ表示。Google Maps で地点名を検索します（別タブ）。

🗑

削除

その地点を削除します（確認あり）。削除後は元に戻せません。

2-6 並び替え（ドラッグ）

地点の長押しドラッグ（スマートフォン）

地点左端の ⠿ ハンドルを長押しすると地点がゴーストコピーになり、ドラッグで順序を変更できます。他の地点の上でドロップすると入れ替わります。

マウスドラッグ（PC）

同じく ⠿ ハンドルをマウスでドラッグして並び替えできます。

注意：地点の編集中（フォームに読み込み中）は並び替えができません。先にキャンセルまたは更新してください。
2-7 帰宅後の記録入力の流れ
1

📂 行程ファイルを読み込む

ツーリング前に保存したJSONファイルを読み込みます。

2

✏️ 編集モードで各地点を開く

地点をタップ → 「✏️ 編集」→ 詳細セクションを開いて実着・実発を入力。走行メモも入力。「更新」で保存。

3

📝 ボタンが amber 点灯したら

実着・実発・走行メモのいずれかが入力されるとボタンが点灯します。

4

📝 走行記録を保存する

ボタンを押すと日付入りの .txt ファイルが保存されます。計画時刻・実績時刻・差分が一覧形式で出力されます。

Chapter 3

走行モードの使いかた

3-1 走行モードとは

🏍️ ボタンで走行モードに切り替えます。行程表示が大型カードに切り替わり、走行中に必要な情報（現在地・天気・出発時刻・次の目的地）が一画面に集約されます。

重要：走行中はスマートフォンを操作しないでください。必ず停車・安全な場所で操作してください。本ツールの操作に起因する事故について制作者は責任を負いません。
縦向き固定：Android Chrome では走行モード開始時に縦向きが自動ロックされます（iOS Safari は非対応）。

▶ 現在地

現在地カード（amber 枠）

現在地として設定した地点。到着・出発時刻、天気、カウントダウン、給油フラグ、メモ、走行メモが大きく表示されます。

あと○分

出発カウントダウン

発時刻が設定されている場合に表示。15秒ごとに自動更新。「出発」（±2分以内）、「○分前」、「○分超過」の3状態で表示されます。

↓ 次の目的地

次地点カード（グレー枠）

次の目的地の地点名・到着予定時刻を表示。「あと○時間○分で到着」も表示されます。

移動時間

区間移動時間（色分け）

前の地点の発〜次の地点の着の区間時間。1時間以内 🔵 〜2時間 🟡 2時間超 🔴 で色分け表示されます。

3-3 スワイプ・ナビゲーション

左右スワイプ

走行ビューを左右にスワイプすると前後の地点カードに切り替わります。アニメーション付きでスムーズに遷移します。

◀ 前 / 次 ▶ ボタン

画面下部のスワイプバーのボタンで同様の切り替えができます。現在の地点位置はプログレスドット（● ○ ○）で確認できます。

閲覧中（◎）表示

現在地以外の地点を閲覧しているときは「◎ 閲覧中」または「✓ 通過済み」タグが表示されます。カードをタップすると「ここを現在地にする」ボタンが出現します。

3-4 現在地の変更・アクション
📍

ここを現在地にする

カードをタップしてアクションを展開 → 「📍 ここを現在地にする」で現在地を変更できます。次の地点に着いたときに使います。

🗺

ルートを開く / マップで確認

日程にルートURLが設定されている場合「🗺 ルートを開く」、地点に住所が設定されている場合「🗺 マップで確認」が表示されます。

✏️

編集モードに戻る

ヘッダーの ✏️ ボタンで走行モードを終了して編集モードに切り替えます。閲覧していた地点の日程に自動でジャンプします。

3-5 天気表示と再取得
🌤️
晴れ
最高 24°C / 最低 12°C　降水 5%

住所が登録されている地点は天気予報が自動取得されます。

🌐 取得中…

取得中は「🌐 取得中…」が表示されます。通信状況により数秒〜10秒かかる場合があります。

⚠️ 再取得（タップ）

取得に失敗すると「⚠️ 再取得」が表示されます。タップすると再試行します。オフライン環境では取得できません。

自動更新：走行モード切り替え時・タブ復帰時・30分ごとに天気が自動更新されます。天気データは当日分のみ表示されます（日付未設定の地点は今日の天気）。

Chapter 4

行程テキスト共有

4-1 この機能について

📋 共有テキストとは

行程全体を読みやすいテキスト形式に変換します。LINE・メモ・メール等に貼り付けて仲間と共有したり、バックアップとして保管するのに便利です。

JSON との違い：JSON ファイルは機械可読な計画データで次回ツーリングに再利用できます。共有テキストは人が読みやすい形式で、再読み込みはできません。

4-3 操作方法
1

📋 ボタンをタップ

全日程のテキストがモーダル画面に表示されます。

2

「クリップボードにコピー」をタップ

テキストがコピーされ、1.2秒後にモーダルが自動で閉じます。

3

LINE・メモ等に貼り付ける

コピーしたテキストを任意のアプリに貼り付けて利用してください。

閉じる方法：✕ ボタン、またはモーダル背景（暗い部分）をタップで閉じます。

Chapter 5

設定画面の使いかた

5-1 スプラッシュ演出の設定

📜 ボタンで設定画面を開きます。起動時に表示される「木枯らし紋次郎」風スプラッシュのカスタマイズができます。

主演名

スプラッシュの主演クレジット（最大20文字）

空欄のとき「あなた」として表示されます。例：田中太郎

第◯話

話数（1〜99）

「第◯話」として表示されます（第一話〜第九十九話）。空欄にするとランダムな話数が表示されます。

タイトル

サブタイトル（最大40文字）

スプラッシュのサブタイトル。空欄にすると「甲州街道に月を追う」のような紋次郎風タイトルがランダム表示されます。

設定するとこのように表示されます：
第十二話
旅刻
主演：田中太郎
甲州街道に月を追う
5-2 デバッグログ

不具合が発生したときに操作履歴・エラー情報を記録・取り出す機能です。通常は OFF のままで問題ありません。

☑️

操作・エラーを記録する（ON/OFF）

チェックするとログ記録を開始。ONにすると画面右下に 🐛 バッジが表示されます。エラーが発生すると赤くなります。

📋

コピー

記録されたログをクリップボードにコピーします。不具合報告の際にそのまま貼り付けて送れます。

💾

保存

ログをテキストファイル（.txt）として保存します。

🗑️

消去

記録されたログをすべて削除します（確認あり）。

不具合が起きたときの手順
① 「操作・エラーを記録する」をON
② 不具合を再現する操作を行う
③「コピー」または「保存」でログを取り出す
④ 報告先にログを貼り付けて送る
注意：デバッグログをONにしたまま長時間使い続けるとログが蓄積されます（最大500件）。不具合確認後はOFFにするか「消去」してください。

6-1 著作権

本ツールの著作権

旅刻mk13（以下「本ツール」）のソースコード・デザイン・UIを含む一切の著作権は、制作者（鴨吉）に帰属します。

本ツールは個人利用を目的として制作されたものです。無断転載・再配布・商用利用・改変した上での公開はご遠慮ください。

利用データの著作権

本ツールに入力した行程データ（地点名・メモ・時刻・走行記録等）の著作権は、入力したご本人に帰属します。本ツールがこれらのデータを外部に送信・収集することはありません。

外部サービスの利用

本ツールは以下の外部サービスを利用しています。各サービスの著作権および利用規約はそれぞれの提供元に帰属します。

• Open-Meteo　天気予報データの取得
• OpenStreetMap Nominatim　住所→座標変換
• 国土地理院 ジオコーダAPI　住所→座標変換（補完）
• OpenStreetMap Overpass API　SA・PA・道の駅データ取得
• Google Maps　マップリンク表示
• Google Fonts　フォントデータ（Noto Sans JP 等）
6-2 免責事項

⚠ 安全運転に関する免責

本ツールは走行計画の補助を目的としたものです。走行中の操作は絶対に行わないでください。

本ツールの使用に起因する事故・違反・損害について、制作者は一切の責任を負いません。交通ルールを遵守し、安全運転に努めてください。

⚠ データ損失に関する免責

本ツールのデータはブラウザの localStorage に保存されます。以下の操作によりデータが消失する場合があります。

• ブラウザの「閲覧データの消去」
• プライベートブラウズ（シークレットモード）の終了
• ブラウザのアンインストール・再インストール
• ブラウザのストレージ上限超過

大切な行程データは必ず 💾 保存ボタン で JSON ファイルとしてバックアップを取ってください。

天気予報・外部データに関する免責

天気予報・施設情報（SA・PA・道の駅等）は外部サービスのデータを利用しています。情報の正確性・最新性を保証するものではなく、これらの情報に基づく判断で生じた損害について制作者は責任を負いません。

サービス継続性に関する免責

本ツールは予告なく機能変更・公開停止される場合があります。また外部 API の仕様変更・サービス停止により、一部機能が利用できなくなる場合があります。

データの外部送信なし
入力した行程データ・メモ・個人情報は外部サーバーに送信されません。すべてのデータはお使いのデバイスのブラウザ内にのみ保存されます。
